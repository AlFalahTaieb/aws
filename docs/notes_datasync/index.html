DataSync

I. Introduction#

  1. AWS DataSync is an online data transfer service that simplifies, automates, and accelerates copying large amounts of data to and from AWS storage services over the internet or AWS Direct Connect.
  2. Can copy between NFS, SMB, S3, EFS and FSx for Windows.
  3. Allows you to copy millions of files without having to build custom solutions or licenses.
  4. You can schedule copy jobs and pass filters to only include the desired files.
  5. Replication tasks can be scheduled hourly, daily, weekly.
  6. Need to install DataSync agent (an EC2 instance running DataSync Agent AMI) on-premise to initiate the transfer.7. Performs integrity checks to make sure the the data is copied correctly to the destination.
  7. You can see the status of the data being copied using CloudWatch metrics.
  8. Preserves directory structure when copying data.
  9. Can use either Direct Connect OR private VPC endpoint (PrivateLink) OR regular internet.
  10. When using VPC endpoint to transfer, data does not traverse public internet.
  11. To use VPC endpoints with AWS DataSync, you create an AWS PrivateLink interface VPC endpoint for the DataSync service in your chosen VPC, and then choose this endpoint elastic network interface (ENI) when creating your DataSync agent. Your agent will connect to this ENI to activate, and subsequently all data transferred by the agent will remain within your configured VPC.
  12. If a task is interrupted, the agent restarts and resumes from previous location.
  13. AWS DataSync assumes an IAM role that you provide. The policy you attach to the role determines which actions the role can perform.
  14. A single DataSync agent is capable of fully-utilizing a 10 Gbps network link.
  15. All data transferred between the source and destination is encrypted via Transport Layer Security (TLS), which replaced Secure Sockets Layer (SSL). Data is never persisted in AWS DataSync itself.
  16. It is PCI compliant and HIPAA eligible.
  17. NOT suited for bandwidth-constrained customers (use snowball or snowcone).
  18. You are only charged for the data you move.
  19. Can transfer directly to S3 Glacier Deep Archive. No need to go through S3 ‚Üí Glacier.

II. DataSync vs Storage Gateway#

DataSync is for transferring while Storage Gateway is for using both AWS and on-prem data stores simultaneously. You can use them together to minimize costs.

Use AWS DataSync to migrate existing data to Amazon S3, and then use the File Gateway configuration of¬†AWS Storage Gateway¬†to retain access to the migrated data and for ongoing updates from your on-premises file-based applications.

III. DataSync vs Snowball#

AWS DataSync is ideal for online data transfers. You can use DataSync to migrate active data to AWS, transfer data to the cloud for analysis and processing, archive data to free up on-premises storage capacity, or replicate data to AWS for business continuity.

AWS Snowball Edge¬†is suitable for offline data transfers, for customers who are bandwidth constrained, or transferring data from remote, disconnected, or austere environments.

IV. DataSync vs S3 Transfer Acceleration#

DataSync loses this round only if you already have your apps integrated with S3 API.
